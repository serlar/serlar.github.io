Detailed Description

Element-wise addition.

The difference between eWiseAdd and eWiseMult is not about the element-wise operation but how the index sets are treated. eWiseAdd returns an object whose indices are the "union" of the indices of the inputs whereas eWiseMult returns an object whose indices are the "intersection" of the indices of the inputs. In both cases, the passed semiring, monoid, or operator operates on the set of values from the resulting index set.

Perform element-wise (general) addition on the elements of two vectors, producing a third vector as result.

Parameters
[in,out]wAn existing OpenGraphBLAS vector. On input, the vector provides values that may be accumulated with the result of the element-wise operation. On output, this vector holds the results of the operation.
[in]maskAn optional "write" mask that controls which results from this operation are stored into the output vector w. The mask dimensions must match those of the vector w. If the GrB_STRUCTURE descriptor is not set for the mask, the domain of the mask vector must be of type bool or any of the predefined "built-in" types GrB_Type. If the default mask is desired (i.e., a mask that is all true with the dimensions of w), GrB_Vector_NULL should be specified.
accumAn optional binary operator used for accumulating entries into existing w entries. If assignment rather than accumulation is desired, GrB_BINARYOP_NULL should be specified.
[in]opThe binary operator used in the element-wise "sum" operation.
[in]uThe OpenGraphBLAS vector holding the values for the left-hand vector in the operation.
[in]vThe OpenGraphBLAS vector holding the values for the right-hand vector in the operation.
[in]descAn optional operation descriptor. If the description is not needed to assist , GrB_DESC_NULL should be specified.
